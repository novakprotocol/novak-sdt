S.D.T. — Software Digital Thread

A small tool and operating model for creating or baselining repos so the next human or AI can understand what the repo is, what is real now, and what to do next.

View repo Releases Proofs Docs

What this is

SDT is a repo birth and repo baseline system. It adds a minimum continuity floor so serious repos stop depending on tribal memory, hidden chat context, or one operator carrying the whole project in their head.

Birth new repos

Start a fresh repo with the operator floor, product truth floor, and status surfaces already in place.

Baseline existing repos

Inspect a repo, show what is missing, and add the minimum survival structure without pretending to invent missing product truth.

Generate reports

Produce Markdown and HTML repo reports that show floor coverage, governance surfaces, discovered docs, and current repo state.

How it works

  1. Clone and install novak-sdt
  2. Run sdt new, sdt baseline, or sdt doctor
  3. Generate a repo report and use the repo with less hidden context
git clone https://github.com/novakprotocol/novak-sdt.git
cd novak-sdt
python3 -m venv .venv
. .venv/bin/activate
python3 -m pip install -e .

sdt baseline   --path /path/to/existing-repo   --product-name "Existing Product"   --product-statement "Existing Product is a product for X."
