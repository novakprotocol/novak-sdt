# SDT C4 context view

## System
SDT is a software digital thread system for repo birth, repo baseline, floor verification, operator handoff, and estate/history/status surfaces.

## Primary actors
- operator
- maintainer
- reviewer
- future human
- future AI

## External systems
- GitHub
- systemd
- local host filesystem
- target repos being born or adopted

## Boundary
SDT owns repo scaffolding, verification, and evidence surfaces.
It does not claim to own every production runtime or notification topology.
