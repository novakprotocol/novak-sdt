# Case Study

## Situation

A repo without strong continuity surfaces tends to hide critical context in:
- scattered markdown
- issue threads
- commit messages
- operator memory
- chat history

## What SDT adds

SDT adds:
- repo identity and current truth surfaces
- operator handoff surfaces
- product truth surfaces
- metadata/governance surfaces
- generated report outputs

## Why that matters

This reduces:
- handoff loss
- cold-start confusion
- repo drift
- hidden assumptions for humans and AI

## What to prove in a real case

The strongest case study is not theory.
It is a real messy repo before baseline, then after baseline, with:
- visible floor files
- visible report outputs
- visible next-action clarity
- reduced hidden context
