# SDT documentation operating model

## Document classes
- tutorial
- how-to
- reference
- explanation
- runbook
- handoff
- status
- evidence
- release

## Workflow
- docs live in git
- docs are reviewed like code
- generated docs are clearly named
- operator docs stay practical
- release docs must state what is deferred

## Required operational metadata
Every operational status doc should carry:
- owner
- last verified
- source of truth
- next review when relevant

## Claims discipline
Documentation must separate:
- proven
- likely
- assumed
- unproven
