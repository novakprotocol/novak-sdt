# SDT 101

## Purpose
This is the shortest practical explanation of what S.D.T. is and what this repo does.

## Git
Git tracks file changes over time.

It gives you:
- commit history
- branches
- rollback points
- change tracking

## GitHub
GitHub hosts Git repos online.

It gives you:
- remote hosting
- sharing
- pull requests
- public visibility
- collaboration

## Repo
A repo is the project folder Git tracks.

A serious repo should not only contain code.
It should also contain:
- current truth
- next actions
- handoff guidance
- product definition

## What S.D.T. is
S.D.T. means Software Digital Thread.

In plain English:
it is a reusable structure and toolset for making repos easier to understand, easier to hand off, and harder to lose.

## What this repo does
This public repo provides a CLI called `sdt`.

It can:
- create a new repo with SDT floors
- baseline an existing repo with SDT floors
- check whether required SDT floor files are present

## Operator floor
The operator floor is the minimum handoff and continuity layer.

It includes:
- `WHAT_IS_REAL_NOW.md`
- `PROJECT_STATE.md`
- `docs/operator/ZERO_CONTEXT_HANDOFF_CHECKLIST.md`
- `docs/operator/COLD_START_RECOVERY.md`
- `docs/operator/NEXT_OPERATOR_PACKET_TEMPLATE.md`

## Product truth floor
The product truth floor is the minimum product-definition layer.

It includes:
- `docs/product/PRODUCT_STATEMENT.md`
- `docs/product/WHAT_THIS_PRODUCT_IS.md`
- `docs/product/CURRENT_VS_PLANNED.md`
- `docs/product/PRODUCT_SYSTEM_BOUNDARY.md`

## Final simple explanation
Git tracks changes.
GitHub hosts repos.
S.D.T. gives repos a minimum survival structure so humans and AI can understand them more easily.
